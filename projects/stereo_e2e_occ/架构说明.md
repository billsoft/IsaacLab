# StereoOccNet 双目立体视觉体素占用网络 — 架构详解

## 一、网络总览：它要解决什么问题？

想象你站在一栋楼的天台往下看，手里拿着两台鱼眼相机（左眼和右眼，间距 8cm）。你想知道下面地面上每一小块（10cm × 10cm × 10cm）空间里有什么东西——是地面、车辆、行人、还是空气。

这就是 **StereoOccNet** 要做的事：

> **输入**：两张鱼眼 RAW 照片（左眼 + 右眼）  
> **输出**：一个 3D 方块网格（72 × 60 × 32），每个小方块标注它属于 18 个语义类别中的哪一个

这个 3D 网格叫做 **体素（Voxel）**，覆盖相机下方 7.2m × 6.0m × 3.2m 的空间，分辨率 0.1m。

---

## 二、整体管线（Pipeline）

```
双目 RAW 图像 [B, 2, 1, 1080, 1280]
       │
       ▼
  ┌─────────────────┐
  │ StereoPatchEmbed │  RAW → 特征图，stride=16
  └────────┬────────┘
           │  [B, 2, 128, 68, 80]
           ▼
  ┌─────────────────┐
  │  ImageEncoder    │  窗口注意力 + 鱼眼射线编码
  └────────┬────────┘
           │  [B, 2, 128, 68, 80]
           ▼
  ┌─────────────────┐
  │ OccupancyDecoder │  粗→精 两阶段解码
  └────────┬────────┘
           │  [B, 36, 30, 16, 128]
           ▼
  ┌─────────────────┐
  │    VoxelHead     │  上采样 + 分类
  └────────┬────────┘
           │  [B, 18, 72, 60, 32]
           ▼
      argmax → 预测体素 [B, 72, 60, 32]
```

下面逐模块深入讲解。

## 三、模块 1：StereoPatchEmbed（RAW 图像嵌入）

### 3.1 它解决什么问题？

普通相机输出 RGB 三通道图像，但这里输入的是 **12-bit RAW Bayer** 单通道图像。Bayer 是感光芯片的原始格式——每个像素只感应红、绿或蓝中的一种颜色，按 RGGB 棋盘格排列。我们需要把这个"生肉"变成神经网络能理解的"特征图"。

### 3.2 内部结构

**第一步：可学习 RGGB 解包**

```
输入: [1, 1080, 1280]  （单通道 RAW）
      ↓ Conv2d(1→4, kernel=2×2, stride=2)
输出: [4, 540, 640]    （4 通道，类似 RGGB 解包）
```

这里用一个 2×2 卷积核，stride=2，效果等价于把相邻 2×2 像素拆成 4 个通道。但权重是可学习的，所以网络可以自己决定怎么组合这些颜色信息——比固定的插值方法更灵活。

**类比**：就像把一本杂志的彩色页面用放大镜看，每 4 个点（红绿绿蓝）组成一个"超级像素"，网络学习如何最好地理解这 4 个点的关系。

**第二步：三级 Stem（逐步降采样）**

```
[4, 540, 640]
  ↓ Conv(4→32, s=2) + BN + GELU
[32, 270, 320]
  ↓ Conv(32→64, s=2) + BN + GELU
[64, 135, 160]
  ↓ Conv(64→128, s=2) + BN + GELU
[128, 68, 80]
  ↓ Conv(128→128, s=1) + BN
[128, 68, 80]
```

经过 RGGB 解包（÷2）和三级卷积（各÷2），原图被缩小了 **16 倍**：1080→68, 1280→80。

**第三步：LayerNorm**

对通道维度做归一化，稳定训练。

### 3.3 输入输出汇总

| 项目 | 维度 |
|------|------|
| 输入 | `[B, 2, 1, 1080, 1280]` — B 张图、2 个相机、单通道 RAW |
| 输出 | `[B, 2, 128, 68, 80]` — 每个相机一张 128 维特征图 |

### 3.4 为什么不直接用 ResNet？

因为输入是 RAW 单通道，不是 RGB。ResNet 的第一层期望 3 通道输入。用可学习的 RGGB 解包 + stem，网络可以直接从传感器原始数据学习，避免了人工 demosaic（去马赛克）的信息损失。

## 四、模块 2：ImageEncoder（图像编码器）

### 4.1 它解决什么问题？

PatchEmbed 产出的特征图是"局部"的——每个位置只看到了周围很小的区域。ImageEncoder 让特征图里的不同位置互相交流信息，同时注入"这个像素对应空间中哪个方向"的几何先验。

### 4.2 内部结构

#### 4.2.1 FisheyeRayEncoding（鱼眼射线编码）

**核心思想**：告诉网络"特征图上的每个点，在真实世界中指向哪个方向"。

**算法原理（初中生版）**：

想象你站在球场中央，手持一个超广角鱼眼镜头朝地面拍照。照片中心对应正下方，照片边缘对应远处。但这不是简单的"越远越偏"——鱼眼镜头有一个特殊的数学规则：

> **f-theta 等距投影**：图像上离中心的距离 r = f × θ

其中 f 是焦距（648 像素），θ 是光线和镜头光轴的夹角。这意味着：角度和图像距离成正比。

**反投影过程**（从图像点推算空间方向）：

1. 计算像素到图像中心的距离：`r = √((u-cx)² + (v-cy)²)`
2. 计算方位角：`φ = atan2(v-cy, u-cx)`
3. 反算入射角：`θ = r / f`
4. 转换为 3D 方向向量：`(sinθ·cosφ, sinθ·sinφ, cosθ)`
5. 用相机外参旋转到世界坐标系

然后用 **正弦位置编码** 把这个 3D 方向向量编码成高维特征，通过 MLP 投射到 128 维，加到图像特征上。

**类比**：给照片上的每个点贴一个"方向标签"——"我指向东北偏下 30°"。这样网络不仅知道"这里长什么样"，还知道"这里对应空间中的哪个方向"。

#### 4.2.2 WindowAttention（窗口自注意力）

将特征图切成 7×7 的小窗口，每个窗口内部做自注意力——让 49 个位置互相交流。

**为什么不用全局注意力？** 因为特征图有 68×80 = 5440 个位置，全局注意力的计算量是 5440² ≈ 3000 万，太贵了。窗口注意力只需要 49² × (5440/49) ≈ 27 万，便宜 100 倍。

#### 4.2.3 EncoderBlock

每个 Block 包含：
- LayerNorm → WindowAttention → 残差连接
- LayerNorm → MLP（扩展 4 倍再缩回来）→ 残差连接

堆叠 2 个 Block（`encoder_layers=2`）。

#### 4.2.4 相机并行处理

两个相机共享同一套 Encoder 参数，通过把 N=2 合并到 batch 维度（`B*N`）一次性处理，避免 Python for 循环。

### 4.3 输入输出汇总

| 项目 | 维度 |
|------|------|
| 输入 | `[B, 2, 128, 68, 80]` + 内参 `[B, 2, 3, 3]` + 外参 `[B, 2, 4, 4]` |
| 输出 | `[B, 2, 128, 68, 80]`（形状不变，但特征被射线编码和注意力增强了） |

## 五、模块 3：OccupancyDecoder（占用解码器）⭐核心

### 5.1 它解决什么问题？

前面的 Encoder 给了我们两张 2D 特征图（左眼、右眼各一张）。但我们要的是 3D 体素。怎么从 2D 变成 3D？

**答案是"可变形交叉注意力"**——让 3D 空间中的每个查询点去 2D 特征图上"偷看"对应位置的信息。

### 5.2 粗→精两阶段策略

直接处理 72×60×32 = 138,240 个体素太多了。所以分两步：

**粗阶段**：先用少量查询（12×10×8 = 960 个）粗略感知整个空间
**精阶段**：上采样到中等密度（36×30×16 = 17,280 个）做精细调整

**类比**：画一幅画时，先用铅笔画个大致轮廓（粗阶段），再用细笔补充细节（精阶段）。

### 5.3 可变形交叉注意力（DeformableCrossAttention）⭐⭐最核心

这是整个网络最关键的组件。它回答一个核心问题：**3D 空间中的某个点，在 2D 图像上的哪个位置？**

#### 5.3.1 参考点投影（get_reference_points）

**步骤拆解**：

1. **归一化坐标 → 世界坐标**

   查询点的坐标是 [0,1] 归一化的。先把它们映射回真实世界的米：
   ```
   x_real = x_norm × (3.6 - (-3.6)) + (-3.6)  → [-3.6m, 3.6m]
   y_real = y_norm × (3.0 - (-3.0)) + (-3.0)  → [-3.0m, 3.0m]
   z_real = z_norm × (2.5 - (-0.7)) + (-0.7)  → [-0.7m, 2.5m]
   ```

2. **世界坐标 → 相机坐标**

   用外参的逆矩阵把世界点变换到相机坐标系：`P_cam = E_inv × P_world`

3. **相机坐标 → 图像坐标（f-theta 鱼眼投影）**

   ```
   r_3d = √(x² + y²)           ← 离光轴的空间距离
   θ = atan2(r_3d, z)           ← 入射角
   φ = atan2(y, x)              ← 方位角
   r_img = f × θ                ← 等距投影：图像距离 = 焦距 × 角度
   u = r_img × cos(φ) + cx      ← 像素列坐标
   v = r_img × sin(φ) + cy      ← 像素行坐标
   ```

4. **归一化到 [-1, 1]** 供 `grid_sample` 使用

5. **FOV 掩码**：相机背面（z < 0）或超出视场角的点标记为无效

**类比（初中生版）**：

想象你在教室黑板前，黑板上贴了一张照片。你想知道教室里某把椅子在照片上的什么位置。你需要：
- 知道椅子在教室里的位置（世界坐标）
- 知道相机放在哪里、朝哪个方向看（外参）
- 知道镜头的放大倍率和畸变规则（内参 + 鱼眼模型）

有了这三个信息，就能算出椅子在照片上的像素位置。

#### 5.3.2 可变形采样

参考点只是一个"大致位置"。网络还会学习 **微小偏移量（offsets）**，在参考点周围采样多个点（默认 4 个），加权求和。

这就像你用手指指向照片上的一个位置说"大概在这里"，然后眼睛自动在周围小范围扫描，找到最有用的信息。

#### 5.3.3 批量 grid_sample 优化

代码把所有相机（N=2）和所有注意力头（num_heads=8）合并到 batch 维度，用 **一次 `grid_sample`** 完成所有采样。这是一个重要的性能优化——原始版本可能需要 N × num_heads = 16 次循环调用。

### 5.4 DeformableDecoderLayer

每层包含：

1. **自注意力**（可选）：让 960 个粗查询互相交流（"你那里看到了什么？"）
2. **可变形交叉注意力**：去 2D 特征图上采样信息
3. **MLP**：非线性特征变换

粗阶段和精阶段各堆叠 2 层（`decoder_layers=2`）。

### 5.5 粗→精过渡

```
粗特征: [B, 128, 12, 10, 8]
    ↓ trilinear 插值
插值结果: [B, 128, 36, 30, 16]
    ↓ coarse_to_fine MLP (128→256→128 + LayerNorm)
精初始查询: [B, 17280, 128]
    ↓ + 3D 正弦位置编码
    ↓ 精阶段 DeformableDecoderLayer × 2
精特征: [B, 36, 30, 16, 128]
    ↓ + Depthwise Conv3d (空间一致性)
输出: [B, 36, 30, 16, 128]
```

### 5.6 外参逆矩阵的解析计算（全链路一致）

代码在所有需要矩阵求逆的地方，都没有使用通用的 `torch.inverse()` 或 `torch.linalg.inv()`，而是统一利用刚体变换的特殊性质做解析求逆：

```
R_inv = R^T        （旋转矩阵的逆 = 转置）
t_inv = -R^T × t   （平移的逆）
```

这一策略在两个位置保持一致：
1. **OccupancyDecoder**：对相机外参求逆，用于可变形交叉注意力的 3D→2D 投影
2. **TemporalFusionModule**：对 ego_motion 矩阵求逆，用于时序记忆对齐

这比通用矩阵求逆更快、数值更稳定（特别是在 FP16 混合精度下），并且完全兼容 TensorRT 部署（`torch.linalg.inv` 在 TensorRT 中不被支持）。

### 5.7 时序融合（可选，默认关闭）

如果启用，会在粗阶段之后加入时序融合模块（TemporalFusionModule），包含三个核心步骤：

#### 5.7.1 Ego-Motion 对齐（align_memory）

上一帧的记忆是在上一帧坐标系下的，但当前帧的相机可能已经移动了。所以需要把旧记忆"搬"到当前坐标系。

**具体做法**：

1. 将粗阶段的 960 个查询 reshape 成 3D 体积 (12×10×8)
2. 在体积上建立 [-1,1] 归一化网格，乘以体素范围的 scale + offset 转换为世界坐标
3. 用 ego_motion 的**解析逆矩阵**（`R_inv = R^T, t_inv = -R^T @ t`，与 Decoder 中相同的刚体求逆方法）将网格从当前帧坐标变换回上一帧坐标
4. 再归一化回 [-1,1]，用 3D `grid_sample` 在旧记忆体积上双线性插值

**为什么用解析求逆而不是 `torch.linalg.inv`？**

ego_motion 是刚体变换（旋转 + 平移），可以用解析公式高效求逆。这样做有两个好处：一是速度更快（跳过了通用矩阵求逆的 O(n³) 计算），二是完全避免了 `torch.linalg.inv` 在 TensorRT 中不被支持的部署兼容性问题。整个网络现在**全链路使用解析刚体求逆**，没有任何 `torch.linalg.inv` 调用。

**类比**：你拍了一段视频，每一帧你站的位置不同。你想把上一帧看到的东西"贴"到当前帧的视角上——就像在地图上把一张照片从旧位置拖到新位置，需要知道你移动了多远、转了多少。

#### 5.7.2 时序注意力（EfficientTemporalAttention）

当前帧特征作为 Query，对齐后的上一帧记忆作为 Key 和 Value，通过 PyTorch 的 `scaled_dot_product_attention`（SDPA）做高效注意力。SDPA 在支持的硬件上会自动启用 FlashAttention 加速。

**意图**：让当前帧有选择性地从历史信息中提取有用的部分——比如某个区域在上一帧被遮挡了，这一帧能看到，网络就不会从历史中取那个区域的信息。

#### 5.7.3 GRU 门控更新（GRUGate）

融合后的特征和旧记忆通过 GRU 门控机制更新：

```
z = σ(W_z · [current, memory])          ← 更新门：决定保留多少新信息
r = σ(W_r · [current, memory])          ← 重置门：决定忘记多少旧信息
h̃ = tanh(W · [current, r ⊙ memory])    ← 候选记忆
new_memory = (1-z) ⊙ memory + z ⊙ h̃    ← 最终记忆
```

**类比**：就像人的短期记忆——看到新东西时，你不会完全忘掉之前看到的，而是有选择地更新。GRU 学习这个"选择性更新"的策略。

### 5.8 输入输出汇总

| 项目 | 维度 |
|------|------|
| 输入 image_feats | `[B, 2, 128, 68, 80]` |
| 输入 intrinsics | `[B, 2, 3, 3]` |
| 输入 extrinsics | `[B, 2, 4, 4]` |
| 输出 voxel_feats | `[B, 36, 30, 16, 128]` |
| 输出 memory | `None`（时序关闭时）或 `[B, 960, 128]` |

## 六、模块 4：VoxelHead（体素输出头）

### 6.1 它解决什么问题？

Decoder 输出的是 36×30×16 的中间特征，但我们需要 72×60×32 的最终预测。VoxelHead 负责上采样和分类。

### 6.2 内部结构

```
输入: [B, 128, 36, 30, 16]  （permute 后）
  ↓ Conv3d(128→64, k=3) + BN + GELU
  ↓ Conv3d(64→32, k=3) + BN + GELU     ← 降通道
[B, 32, 36, 30, 16]
  ↓ trilinear 插值到 (72, 60, 32)        ← 空间上采样
[B, 32, 72, 60, 32]
  ↓ refine: Conv3d(32→32, k=3) + BN + GELU
  ↓ + skip: Conv3d(32→32, k=1)           ← 残差连接
[B, 32, 72, 60, 32]
  ↓ cls_head: Conv3d(32→18, k=3)
  ↓ + cls_skip: Conv3d(32→18, k=1)       ← 双路分类
[B, 18, 72, 60, 32]
```

### 6.3 设计亮点

- **先降通道再上采样**：在低分辨率时降通道（128→32），然后上采样。这比先上采样再降通道省很多计算量
- **双路分类**：3×3 卷积捕获邻域关系 + 1×1 卷积做逐点分类，两路相加。类似 ResNet 的 skip 思想

### 6.4 输入输出汇总

| 项目 | 维度 |
|------|------|
| 输入 | `[B, 36, 30, 16, 128]` |
| 输出 | `[B, 18, 72, 60, 32]` — 18 个类别的 logits |

## 七、损失函数：OccupancyLoss

### 7.1 组成

**CE Loss（交叉熵）**：标准的多类别分类损失。预测越接近真实标签，损失越小。

**Lovász-Softmax Loss**：针对 IoU（交并比）指标优化的损失。交叉熵只关心每个体素独立的分类对错，而 Lovász 关心的是每个类别整体的 IoU——更接近最终评估指标。

```
总损失 = CE + 0.5 × Lovász
```

### 7.2 关键细节

- **ignore_index = 255**：标记为"未观测"的体素不参与损失计算
- **Lovász 强制 FP32**：在混合精度训练下，softmax + sort 操作对精度敏感，强制使用全精度
- **只遍历出现的类**：跳过当前 batch 中不存在的类别，避免无效计算

### 7.3 Lovász-Softmax 算法原理（初中生版）

普通损失函数一次只看一个方块："这个方块猜对了吗？"

Lovász 换了一种问法："我把所有猜测是'车辆'的方块画出来，和真实是'车辆'的方块比较，重叠了多少？" 重叠比例（IoU）越高越好。

但 IoU 不能直接求导（因为涉及"排序"操作），所以 Lovász 用了一个数学技巧：把 IoU 的计算变成一个可以求导的近似版本。具体是把每个像素的误差从大到小排序，然后用 Jaccard 指数的变化量作为权重——误差越大的像素，权重越高。

## 八、数据集：StereoOccDataset

### 8.1 数据目录结构

```
output_dng/
├── calibration.json           ← 相机标定参数
├── voxel_config.json          ← 体素网格配置
├── left_dng/frame_XXXXXX.dng  ← 左眼 RAW 图像
├── right_dng/frame_XXXXXX.dng ← 右眼 RAW 图像
├── voxel/frame_XXXXXX_semantic.npz ← 体素真值标签
├── meta/frame_XXXXXX.json     ← 帧级元数据
├── train.txt                  ← 训练集帧列表
└── val.txt                    ← 验证集帧列表
```

### 8.2 相机模型

**朝下相机**：相机安装在高处，光轴指向地面（-Z 方向）。这在智慧园区/机器人巡检场景中很典型。

**外参构建**：

`_camera_down_rotation()` 方法构建 Camera→World 旋转矩阵，考虑了：
- USD 仿真器中相机默认光轴 = -Z
- 数据采集时 prim 旋转 euler=[0,0,90]
- f-theta 投影约定：光轴 = +Z（OpenCV 风格）

最终效果：f-theta 的 +Z（光轴）对应世界的 -Z（朝下）。

### 8.3 DNG 加载

支持两种库：`tifffile`（首选，轻量）和 `rawpy`（备选）。RAW 值范围 0~4095（12-bit），归一化到 [0, 1]。

### 8.4 时序模式

当 `use_temporal=True` 时，`__getitem__` 返回连续多帧数据，维度多一个时间维度 T。

## 九、训练代码详解

### 9.1 训练策略总览

| 策略 | 实现 |
|------|------|
| 优化器 | AdamW（lr=1e-4, weight_decay=0.01） |
| 学习率调度 | CosineAnnealingLR |
| 混合精度 | `torch.amp.autocast` + `GradScaler`（可选） |
| 梯度累积 | 可配置步数（默认 1） |
| 梯度裁剪 | `clip_grad_norm_(1.0)` |
| 时序训练 | TBPTT（截断式时间反向传播） |
| 梯度检查点 | 精阶段 Decoder 默认开启 |

### 9.2 混合精度训练（AMP）

**原理**：前向传播用 FP16（省显存、快），反向传播的梯度用 FP32（保精度）。`GradScaler` 自动缩放梯度防止 FP16 下溢。

**特殊处理**：Lovász-Softmax 损失强制 FP32，因为 softmax + sort 在 FP16 下精度不够。

### 9.3 梯度累积

当显存不够跑大 batch 时，把梯度累积 N 步再更新一次参数，等效于 batch_size × N。

代码将损失除以 `grad_accum_steps`，每 N 步做一次 `scaler.step(optimizer)`。

### 9.4 TBPTT（截断式时间反向传播）

时序训练时，长序列的反向传播会占用大量显存。TBPTT 把序列切成小块（chunk_size=2），每个块内正常反向传播，块之间 `memory.detach()` 截断梯度。

这是**显存和梯度质量之间的权衡**——完全不截断最准确但可能爆显存，完全截断等于单帧训练。chunk_size=2 是一个折中。

**ego_motion 的计算**：训练循环在模型外部通过 `torch.linalg.inv(pose_t) @ pose_prev` 计算两帧之间的相对运动，作为输入传给模型。注意这个 `linalg.inv` 调用在训练脚本中而非模型内部，不会影响 ONNX/TensorRT 导出——部署时 ego_motion 由外部系统（如 IMU/里程计）提供。

### 9.5 时间加权

时序训练中，后面帧的损失权重更高：`weight = 1.0 + t / (T-1)`。

**意图**：后面帧已经积累了时序信息，应该预测得更准，给更高权重促使网络充分利用历史信息。

### 9.6 梯度检查点（Gradient Checkpointing）

精阶段 Decoder 默认开启 `torch.utils.checkpoint`。前向传播不保存中间激活值，反向传播时重新计算。**用时间换显存**——精阶段有 17,280 个查询，中间激活占显存最大。

### 9.7 验证与保存

- 每个 epoch 结束后在验证集上评估
- 保存验证损失最低的模型为 `best_model.pth`
- Checkpoint 包含完整状态：model、optimizer、scheduler、scaler、config，支持断点续训

## 十、位置编码系统

### 10.1 SineCosinePositionEncoding3D

为 3D 体素网格生成位置编码。将 (x, y, z) 坐标分别编码为正弦和余弦信号，然后拼接。

**原理**：不同频率的正弦/余弦函数可以唯一编码任何位置。低频捕获大范围位置关系，高频捕获精细位置差异。维度分配：dim/6 给 x-sin, dim/6 给 x-cos, 同样给 y 和 z，共 dim 维。

### 10.2 FisheyeRayEncoding

已在 ImageEncoder 部分详述。核心是 f-theta 反投影 + 正弦编码 + MLP。

## 十四、核心算法原理总结

### 14.1 从 2D 到 3D 的桥梁

整个网络最核心的创新在于：**用可变形交叉注意力作为 2D→3D 的桥梁**。

传统方法（如 LSS/BEVDet）是先估计深度分布，把 2D 特征"提升"到 3D 空间。而这个网络用的是 **查询式方法**：

1. 在 3D 空间预设一组查询点
2. 计算每个查询点在图像上的投影位置
3. 从图像特征上采样该位置的信息
4. 查询点吸收这些信息后，就获得了对应 3D 位置的语义理解

这种方法不需要显式的深度估计，更加端到端。

### 14.2 双目立体信息的利用

虽然代码中没有显式的"立体匹配"模块，但双目信息通过两种方式被隐式利用：

1. **同一 3D 查询点投影到左右眼不同位置**：网络通过交叉注意力从两个视角采样，天然获得了视差信息
2. **注意力权重学习**：网络可以学会在遮挡时依赖一个视角，无遮挡时融合两个视角

### 14.3 鱼眼相机的特殊处理

整个网络针对 f-theta 鱼眼相机做了专门适配：

- 投影模型：`r = f × θ`（不是针孔的 `r = f × tan(θ)`）
- 射线编码：基于 f-theta 反投影
- FOV 掩码：超出 157.2° 视场角的投影点被屏蔽
