## 第 1 项：grid_sample 逐头循环 → 批量调用

**文件**: `deformable_attention.py` 第 129-148 行

**现状问题**:
```python
for cam in range(N):        # N=2
    for h in range(self.num_heads):  # 8
        v_h = v_cam[..., h, :].permute(0, 3, 1, 2)  # [B, head_dim, H, W]
        locs_h = locs[:, :, h, :].contiguous().view(B, Q*num_points, 1, 2)
        s_h = F.grid_sample(v_h, locs_h, ...)        # 逐头调用
```
每次前向传播 2×8=16 次 grid_sample，decoder 4 层 = **64 次调用**。

**修复方案**:
将 head 维度合并到 batch 维度，每相机只调一次 grid_sample：
```python
for cam in range(N):
    # value: [B, H, W, num_heads, head_dim] → [B*num_heads, head_dim, H, W]
    v_cam = value_proj[:, cam]
    v_batched = v_cam.permute(0, 3, 4, 1, 2).reshape(B * num_heads, head_dim, H, W)
    # locs: [B, Q, num_heads, num_points, 2] → [B*num_heads, Q*num_points, 1, 2]
    locs_batched = locs.permute(0, 2, 1, 3, 4).reshape(B * num_heads, Q * num_points, 1, 2)
    # 单次 grid_sample
    sampled = F.grid_sample(v_batched, locs_batched, ...)  # [B*num_heads, head_dim, Q*P, 1]
    sampled = sampled.view(B, num_heads, head_dim, Q, num_points)
```
从 16 次降至 **2 次** grid_sample（每相机 1 次），减少 87.5% 的 kernel launch 开销。

## 第 3 项：固定计算缓存

### 3a. `SineCosinePositionEncoding3D.forward` 重复计算

**文件**: `position_encoding.py` 第 22-50 行
**调用处**: `occ_decoder.py` 第 106 行和第 133 行

**现状**: 每次 forward 都重新 `torch.arange` + `sin/cos`，但粗/精尺寸 (12,10,8) 和 (36,30,16) 是固定的。

**修复方案**: 在 `OccupancyDecoder.__init__` 中预计算，存为 `register_buffer`：
```python
# __init__ 中
self.register_buffer('coarse_pos', self.pos_3d(cx, cy, cz, 'cpu'))
self.register_buffer('fine_pos', self.pos_3d(fx, fy, fz, 'cpu'))

# forward 中直接用
query = self.coarse_query.expand(B, -1, -1) + self.coarse_pos.unsqueeze(0)
```
之后 `pos_3d` 模块可设为仅 init 时使用，不再每帧调用。

### 3b. `torch.inverse(extrinsics)` 每帧重算

**文件**: `deformable_attention.py` 第 65 行

**现状**: `inv_extrinsics = torch.inverse(extrinsics)` 每层每次前向都调用。4 层 decoder = 4 次矩阵求逆。

**修复方案**: 在 `OccupancyDecoder.forward` 开头预计算一次，传入各层：
```python
inv_extrinsics = torch.inverse(extrinsics)
# 传给 DeformableCrossAttention
```
需要修改 `DeformableDecoderLayer.forward` 和 `DeformableCrossAttention.forward` 的签名，增加 `inv_extrinsics` 参数。

### 3c. `FisheyeRayEncoding.get_rays_from_params` 中的 meshgrid

**文件**: `position_encoding.py` 第 100-104 行

**现状**: 每次调用生成 68×80 的像素网格。修复 #4 后坐标会变为原始分辨率采样点，但仍然是固定的。

**修复方案**: 在 `__init__` 中预计算并 `register_buffer`：
```python
# __init__ 中
v, u = self._build_pixel_grid(image_size, feat_size)
self.register_buffer('u_coords', u)
self.register_buffer('v_coords', v)
```

## 第 4 项：FisheyeRayEncoding 坐标系不匹配 ⚠️ 关键正确性 Bug

**文件**: `position_encoding.py` 第 87-117 行

**现状问题**:
```python
def get_rays_from_params(self, intrinsics, extrinsics, H, W):
    # H=68, W=80 (特征图尺寸)
    v_coords, u_coords = torch.meshgrid(
        torch.linspace(0, H - 1, H, device=device),  # [0, 67]
        torch.linspace(0, W - 1, W, device=device),   # [0, 79]
    )
    # 内参 cx=640, cy=540 是原始 1280×1080 分辨率的
    dx = u_coords - cx  # 全部 < 0！因为 u ∈ [0,79], cx=640
    dy = v_coords - cy  # 全部 < 0！因为 v ∈ [0,67], cy=540
```
像素坐标范围 [0,79]×[0,67] 与内参光心 (640, 540) 完全不匹配，导致所有射线指向相同象限，编码信息严重退化。

**修复方案**:

在原图分辨率 [0, 1279]×[0, 1079] 上均匀采样 80×68 个点：
```python
# 在原始图像分辨率上均匀采样 feat_size 个点
orig_H, orig_W = self.image_size  # (1080, 1280)
v_coords, u_coords = torch.meshgrid(
    torch.linspace(0, orig_H - 1, H, device=device),  # 68 个点在 [0, 1079]
    torch.linspace(0, orig_W - 1, W, device=device),   # 80 个点在 [0, 1279]
    indexing='ij',
)
```
这样 dx = u - 640 ∈ [-640, +639]，dy = v - 540 ∈ [-540, +539]，射线覆盖整个 FOV。

同时修复 `deformable_attention.py` 第 91-92 行的归一化：
```python
# 当前用 H, W (特征图尺寸) 归一化
u_norm = 2.0 * u / (W - 1) - 1.0   # W=80
v_norm = 2.0 * v / (H - 1) - 1.0   # H=68
```
这里的 u, v 是从内参 (cx=640) 计算出的像素坐标，值域在 [0, 1279]。
用 W-1=79 归一化是错的，应该用原图尺寸：
```python
orig_H, orig_W = self.config.image_size
u_norm = 2.0 * u / (orig_W - 1) - 1.0   # 1279
v_norm = 2.0 * v / (orig_H - 1) - 1.0   # 1079
```
但由于 `grid_sample` 采样的是特征图 (68×80)，而归一化坐标 [-1,1] 映射到特征图的空间范围，所以这里实际是正确的——只要 get_reference_points 输出的 u,v 在特征图坐标系下即可。

**关键洞察**: `get_reference_points` 中的投影结果 u, v 是原图像素坐标，但 `grid_sample` 采样的是特征图。需要将 u, v 从原图坐标映射到特征图坐标再归一化：
```python
# 原图像素 → 特征图像素
u_feat = u * (W - 1) / (orig_W - 1)
v_feat = v * (H - 1) / (orig_H - 1)
# 特征图像素 → [-1, 1]
u_norm = 2.0 * u_feat / (W - 1) - 1.0
v_norm = 2.0 * v_feat / (H - 1) - 1.0
# 化简后等价于：
u_norm = 2.0 * u / (orig_W - 1) - 1.0
v_norm = 2.0 * v / (orig_H - 1) - 1.0
```

所以 `deformable_attention.py` 的归一化也要改为用原图尺寸。需要从 config 获取 `image_size`。

## 第 5 项：外参旋转矩阵缺失 ⚠️ 关键正确性 Bug

**文件**: `dataset.py` 第 141-174 行

**现状问题**:
```python
# 第 164 行注释说"简化: 使用单位旋转"
E = torch.eye(4).unsqueeze(0).repeat(2, 1, 1)  # 单位矩阵
E[:, 2, 3] = cam_pos[2]    # 只有平移
E[0, 1, 3] = baseline / 2  # 基线偏移
E[1, 1, 3] = -baseline / 2
```
相机实际朝下（euler=[0,0,90]，光轴指向 -Z），但外参是单位旋转矩阵（光轴指向 +Z）。

可变形注意力中 `get_reference_points` 使用外参做 3D→相机坐标投影，旋转矩阵错误导致投影点位置完全错误。

**修复方案**:

Isaac Sim 相机坐标系：+X=前(光轴), +Y=左, +Z=上。
USD prim `euler=[0,0,90]` 使相机朝下，即 +X 指向世界 -Z。

Camera→World 旋转矩阵（绕 X 轴 90°，使光轴 +X 映射到世界 -Z）：
```python
# 相机坐标 → 世界坐标的旋转
# 相机 +X(光轴) → 世界 -Z(向下)
# 相机 +Y(左)   → 世界 +Y(左)
# 相机 +Z(上)   → 世界 +X(前)
R_cam2world = torch.tensor([
    [0, 0, 1],   # 世界 X = 相机 Z
    [0, 1, 0],   # 世界 Y = 相机 Y
    [-1, 0, 0],  # 世界 Z = -相机 X
], dtype=torch.float32)
```

完整外参矩阵：
```python
E = torch.eye(4).unsqueeze(0).repeat(2, 1, 1)
E[:, :3, :3] = R_cam2world
E[:, 2, 3] = cam_pos[2]
E[0, 1, 3] = baseline / 2
E[1, 1, 3] = -baseline / 2
```

同时 `_build_camera_matrices` (第 68-96 行) 也需要同步修改。

## 第 6 项：Lovász 损失排序优化

**文件**: `loss.py` 第 40-66 行

**现状问题**:
```python
for c in range(C):  # C=18
    fg = (target_flat == c).float()
    if fg.sum() == 0:
        continue
    errors = (fg - prob_flat[c]).abs()
    errors_sorted, perm = torch.sort(errors, descending=True)  # ~138K 元素排序
```
最多 18 次 torch.sort，每次排 ~138,000 (72×60×32) 个元素。

**修复方案**:

1. 跳过空类更积极 —— 统计前先收集存在的类：
```python
present_classes = target_flat.unique()
for c in present_classes:
    ...
```

2. 对小体素网格（138K）影响有限，但若未来扩大可考虑 Top-K 近似。当前先做跳过空类优化。

## 第 7 项：TensorRT / ONNX 导出路径（暂不修复）

**问题**: `torch.inverse`、Python 循环、`F.grid_sample` 的 `align_corners=False`、条件分支等对 TRT 不友好。

**计划**: 先确保训练收敛和推理正确，后续专门做导出适配：

- Python 循环 → 批量化（第 1 项已解决）
- grid_sample → TRT 8.6+ 支持
- 条件分支 → 导出时固定 temporal 开关
